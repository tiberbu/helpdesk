# File List Tracking Policy

## What the File List is for

Every story file contains a **File List** section. Its purpose is to enumerate the source-code files a story implementation touches — so a reviewer can verify that the commit scope matches what was declared.

## Excluded categories

The following categories are **auto-generated** or **tooling-managed** and are intentionally excluded from File List tracking requirements:

| Category | Path pattern | Reason |
|----------|-------------|--------|
| Story artifact files | `_bmad-output/implementation-artifacts/story-*.md` | Created/updated automatically by the BMAD harness for every task in a session. Including them would require each story to list every other story that happened to be active in the same session commit. |
| Sprint status | `_bmad-output/sprint-status.yaml` | Updated automatically by the BMAD harness on every task transition. |
| QA report files | `docs/qa-report-*.md` | Generated by QA/adversarial-review tasks; the story being reviewed is not responsible for declaring files produced by the reviewing task. |

## Rationale

Without this policy, every fix story that touched even one real source file would also need to enumerate 6–12 `_bmad-output/` and `docs/qa-report-*.md` files bundled in the same commit by the BMAD session harness. This caused a recursive "commit-scope pollution" finding in every subsequent QA review, producing an infinite loop of fix stories each declaring the previous story's auto-generated files.

## What must still be declared

All of the following **must** appear in the File List:

- Production source code (`.py`, `.js`, `.vue`, `.ts`, `.json` DocType definitions, etc.)
- Test files (`test_*.py`)
- Configuration files (`.txt` patches, `.yaml` config, `.gitattributes`, etc.)
- Documentation files the story explicitly modifies (e.g., `docs/architecture.md`)
- The story's own tracking file (self-reference)

## Enforcement

This is a convention, not a hard tool enforcement. Reviewers applying the adversarial-review QA template should skip commit-scope pollution findings for files matching the excluded path patterns above.
